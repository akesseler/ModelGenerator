Overview

The Plexdata Model Generator allows to convert an XML or a JSON string into so–called models. Model means in this context a container to manage data. With such containers, an automated serialization and deserialization becomes possible. In addition, code generation can take place for both C# and Visual Basic.

Naming Rules

There are some rules how classes and their members are named. These naming rules will be briefly discussed here.

All designations are based on the named entries in the XML or JSON sources. These original designations are then reused in the generated code for the corresponding attributes. In addition, these original designations are used for generating class and member names. However, class and member names follow different rules that must be considered accordingly.

Also note, an original designations may start with a digit. However, both class names and member names must never start with digits. Therefore, the character ‘N’ is prepended in such a case.

Required Settings

Some settings are required to allow the Naming Rules to be applied. First of all the setting Enable usage of adjustment feature. must be enabled. Secondly, the value of Casing Format: must be different from Unmodified. See figure below what is meant in detail.

Naming of Classes

As first, all non–letter and non–digit characters are removed from the original name. Then the intermediate result is split at all uppercase letters. Thereafter, each first characters is converted to upper case and all remaining characters are converted to lower case. But please keep in mind, this conversion step also depends on current settings. Finally, all name parts are reassembled.

Examples

SIsUsedForPORT443OnHTTP
This source identifier will be converted into class name SIsUsedForPort443OnHttp.
_S-Is-Used_For_PORT-443-On_HTTP
This source identifier will be converted into class name SIsUsedForPort443OnHttp.
_443IsThePortForSOnHTTP
This source identifier will be converted into class name N443IsThePortForSOnHttp.
The-Same-Source-Name and The_Same_Source_Name
These both source identifiers will be converted into class name TheSameSourceName. These different source identifiers may cause trouble if their members are not the same. Therefore, such constellations should be prevented.

Finally note the special treatment of the very common naming part ‘IDs’. This exact combination of characters is always converted to Ids.

Naming of Members

Basically, the same rules apply to the naming of members (properties or fields) as to the naming of classes. However, additional modifications will take place and these additional modifications as well as other restrictions will be part of this section. See list below for more details.

  • The adjustment feature must be enabled at all. See section Required Settings for more details.
  • The suffix Value is appended if the class name equals one of its child member names. But this can only be the case if no prefix or no suffix is configured.
  • Duplicate member names are extended by a continuous index. As described above this may sometimes happen in case of the original designations result in the same member name.
  • The plural version of the last naming part is chosen if a member could be identified as an array type.
  • The casing format (PascalCase or camelCase) is applied, before a prefix is prepended.
  • Both the prefix and the suffix is taken as they are. This in turn means that each whitespace, each special character, etc. will be part of the resulting member name.
